Encrypted file transfer. Live WebRTC peer-to-peer, or async via the relay mailbox. Open source, MIT.

HOW IT WORKS

01

Encrypt locally

Your file is encrypted in the browser or CLI before any byte leaves your machine. NaCl secretbox, 256-bit key derived from a passphrase via SHA-256 with domain separation. The key never crosses the network.

02

Pair through the relay

Both devices connect to the relay's WebSocket room and exchange WebRTC SDP offer/answer + ICE candidates. The relay is zero-knowledge. It sees ciphertext signaling messages and nothing else.

03

Transfer direct

When the WebRTC DataChannel opens, file bytes flow straight between the two devices, bypassing the relay. If NAT punching fails, both sides fall back to streaming through the relay (still ciphertext).

RELAY SEES → a 16-char hex token · ciphertext blobs · nothing else

USE IT

INSTALL CLI
curl -sSL https://airpipe.sanyamgarg.com/install.sh | sh cp
go install github.com/Sanyam-G/Airpipe/cmd/airpipe@latest cp

Linux + macOS · amd64 + arm64. Update with airpipe update.

SEND A FILE
$ airpipe send report.pdf
  RIVER FALCON MARBLE 42
  Tell them: airpipe.sanyamgarg.com
  They type the code, they get the file.
DOWNLOAD
$ airpipe download RIVER FALCON MARBLE 42

Or paste the passphrase on the homepage. Or scan the QR.

RECEIVE FROM BROWSER (P2P)
$ airpipe receive ./downloads

Prints a QR. Phone scans it, drops a file. WebRTC direct, falling back to relay if NAT punching fails.
